# M0 — Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the Bun + TypeScript project skeleton that compiles to a single binary, runs `styre --version`, and self-bootstraps the SQLite SoT via `styre migrate`.

**Architecture:** A single Bun package. The binary entry (`src/index.ts`) dispatches subcommands via `citty`. `styre migrate` opens the embedded-SQLite database with the daemon's PRAGMAs and applies the canonical schema (`src/db/schema.sql`, ported from `docs/architecture/schema.sql`) if absent — idempotent self-bootstrap (control-loop §10). This is milestone M0 of `docs/brainstorms/2026-06-20-scaffolding-design.md`; later milestones deepen each subsystem.

**Tech Stack:** Bun (runtime, `bun build --compile`, `bun:sqlite`, `bun test`), TypeScript, `citty` (CLI), Biome (lint/format). No Anthropic SDK, no zod yet (YAGNI — added when first used in later milestones).

## Global Constraints

- **Runtime is Bun** — `bun:sqlite` (no native SQLite addon), `Bun.spawn`, `bun test`. (SC-1/SC-2.)
- **Single self-contained binary** — everything must survive `bun build --compile`; embed assets via `with { type: "text" }` / JSON import, never read repo-relative files at runtime.
- **Timestamps stored UTC; displayed local** at the render edge only (DS-1 / CL-INV-8). (No display surfaces in M0, but never store local-tz.)
- **XDG paths on macOS + Linux both** — DB under `$XDG_STATE_HOME/styre/` (build-operations §3.1).
- **Dev workflow (hard rules):** never commit to `main`; work on a `feat/` branch; PR-only merges; **no auto-merge** (operator merges). Use Conventional Commits.
- **TDD throughout** — failing test first, minimal implementation, green, commit. Keep files small and single-responsibility.
- **Schema is canonical at `docs/architecture/schema.sql`** — `src/db/schema.sql` is a verbatim port; keep them identical (later milestones make `src/db/` the migration source of record).

/** Open the SoT DB with the daemon's required PRAGMAs (control-loop §2.2). */

  const db = new Database(path, { create: true });

```ts
import { mkdirSync } from "node:fs";
import { dirname } from "node:path";
import type { Database } from "bun:sqlite";
import schemaSql from "./schema.sql" with { type: "text" };
import { openDb } from "./client.ts";

export interface MigrateResult {
  version: number;
  created: boolean;
}

function readVersion(db: Database): number | null {
  try {
    const row = db
      .query<{ version: number }, []>(
        "SELECT version FROM schema_meta ORDER BY version DESC LIMIT 1",
      )
      .get();
    return row ? row.version : null;
  } catch {
    return null; // schema_meta table absent → fresh DB
  }
}

/** Self-bootstrapping migrate (control-loop §10): create DB + schema if absent; idempotent. */
export function migrate(path: string): MigrateResult {
  mkdirSync(dirname(path), { recursive: true });
  const db = openDb(path);
  try {
    const existing = readVersion(db);
    if (existing !== null) {
      return { version: existing, created: false };
    }
    db.exec(schemaSql);
    const version = readVersion(db);
    if (version === null) {
      throw new Error("migrate: schema_meta empty after bootstrap");
    }
    return { version, created: true };
  } finally {
    db.close();
  }
}
```
